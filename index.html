Proposed Summer Project

LMS Integration Module: Bridging Classrooms and Code

Problem Statement: Right now, App Inventor doesn't fit naturally into how schools already work. Teachers rely on tools like Canvas or Google Classroom, but App Inventor sits outside that workflow. So assignments get messy - files sent over email, versions lost, progress hard to track. That friction quietly discourages adoption.

Proposed Solution: I want to build an LMS integration layer that connects App Inventor directly to platforms like Canvas and Google Classroom. Teachers can create assignments, students can submit projects, and grading happens in one place. No more workarounds, just a smooth classroom experience.

Technical Architecture

Component Technology Purpose
LMS Adapter Interface Java Abstract layer for LMS-specific implementations
Canvas LTI 1.3 Integration Java + REST API Deep linking, assignment creation, grade passback
Google Classroom Integration JavaScript/Node.js microservice OAuth2 flows, CourseWork API interactions
Frontend UI Components GWT Teacher dashboard for assignment management
Submission Handler Java AIA file ingestion, metadata extraction, version tracking

Implementation Timeline (175 hours)

Phase 1: Core Infrastructure (Weeks 1-4, 50 hours)

  • Extend App Inventor's project metadata system to include assignmentId and courseId fields
  • Implement secure OAuth2 token storage per-user (encrypted at rest)
  • Create LMS adapter interface with pluggable architecture for multiple providers
  • Build local testing environment with Canvas sandbox

Phase 2: Canvas Integration (Weeks 5-9, 75 hours)

  • LTI 1.3 Advantage Services for seamless launch from Canvas into App Inventor with context
  • Assignment creation via /api/v1/courses/:course_id/assignments with App Inventor-specific submission types
  • Grade passback via /api/v1/courses/:course_id/assignments/:assignment_id/submissions/:user_id
  • Rubric assessment support for structured grading

Phase 3: Google Classroom Integration (Weeks 10-11, 35 hours)

  • CourseWork API integration for assignment distribution
  • StudentSubmission API for .AIA file collection
  • Simplified OAuth2 flow for K-12 accessibility

Phase 4: UI/UX & Testing (Week 12, 15 hours)

  • Teacher-facing dashboard in Designer view (GWT)
  • Student submission flow with progress indicators
  • End-to-end testing with sandbox LMS instances

Why This Should Be a Priority for App Inventor

  1. Fits how schools already operate: LMS platforms are now standard. If App Inventor doesn't integrate, it risks being left out of formal education.
  2. Better access for students: Cleaner submission flows reduce issues for students with unstable internet or limited resources.
  3. Unlocks learning insights: Structured submissions make it easier to understand how students learn and improve teaching tools over time.
  4. Keeps App Inventor competitive: Other platforms already support classroom workflows. This closes that gap.
